LOCAL DEV
---------
1. Install deps (pnpm is required — the lockfile is pnpm-lock.yaml):
     pnpm install

2. Create .env.local with your Anthropic key (gitignored already):
     echo 'ANTHROPIC_API_KEY=sk-ant-...' > .env.local

3. Start dev server (Turbopack, port 3099 is what we usually use):
     npx next dev --turbopack -p 3099

   Or just `pnpm dev` if you prefer the default port 3000.

4. The dev server picks up .env.local at startup. If you write it AFTER
   the server is already running, restart the server.

DEPLOY (one-off, from your terminal)
------------------------------------
From the repo root or worktree:

ERROR HANDLING
--------------
Both AI routes wrap the Anthropic SDK calls in try/catch and return
upstream errors as JSON so the UI can show actionable messages. If you
ever see a generic "Survey request failed" toast, the underlying message
is now in the response body — most commonly:
  "Your credit balance is too low to access the Anthropic API."
  → top up at https://console.anthropic.com/settings/billing

GOTCHAS / THINGS THAT WILL BITE YOU
-----------------------------------
1. Don't commit .env.local. It's in .gitignore already as `.env*`.
2. Don't commit the test fixture sample-floor-plan.png if you regenerate it.
3. pnpm-lock.yaml is the source of truth. If you ever see a package-lock.json,
   delete it before deploying — Vercel uses pnpm based on the lockfile and
   chokes on the npm one.
4. The /api/ai/survey route uses Claude with vision, so the image must be
   PNG/JPG/WebP/GIF (NOT PDF). The dialog enforces this.
5. The dialog's file input is INSIDE a fixed-position overlay. If you're
   programmatically uploading via DataTransfer, scope your querySelector to
   `.fixed.inset-0` so you don't accidentally hit the toolbar's plan-upload
   input by mistake.
6. The "Untitled design" input in the top bar is editable but only shows a
   pencil icon on hover — that's by design.
